# cyanea-datasets Architecture

## Module Map

```
cyanea-datasets
 +-- genomics.rs       E. coli 16S, SARS-CoV-2 spike (nt + protein), VCF variants, gene annotations
 +-- alignment.rs      Spike variant MSA seqs, hemoglobin cross-species, pre-computed alignment, CIGAR strings
 +-- epigenomics.rs    ChIP-seq narrow peaks, narrowPeak/BED formatting, CpG methylation sites
 +-- single_cell.rs    PBMC 50-cell expression matrix with cell types and QC methods
 +-- chemistry.rs      FDA-approved drug library with SMILES, properties, Lipinski checks, SDF output
 +-- phylogenetics.rs  Primate Newick/NEXUS trees, cytochrome b sequences, FASTA, distance matrix
 +-- metagenomics.rs   Gut microbiome OTU table, relative abundance, Shannon diversity, taxonomy lineages
 +-- structural.rs     Insulin chain B Ca trace, distance/contact map, PDB output, Ramachandran angles
```

## Design Decisions

### In-Memory Only

All datasets are compiled into the binary as `const` byte slices or generated by deterministic functions. There are no external files, no filesystem access, and no network calls. This makes the crate usable in WASM and `no_std`-compatible environments and ensures reproducibility across runs and platforms.

### Static Data vs. Generated Data

Two patterns are used depending on the dataset:

1. **Static constants**: Sequence data (E. coli 16S, SARS-CoV-2 spike, hemoglobin, cytochrome b, insulin coordinates) is stored as `const &[u8]` or `&str` literals. These are compiled directly into the binary's read-only data segment with zero runtime cost.

2. **Builder functions**: Structured datasets (variants, peaks, methylation sites, OTU tables, drug library) are constructed in `Vec`s at call time. This keeps the source readable (one struct literal per record) while allowing methods on the returned types.

The single-cell module uses a third pattern: **deterministic pseudo-random generation**. The expression matrix is generated from a seed-based LCG (`state * 6364136223846793005 + 1442695040888963407`) to produce biologically plausible count data without storing a 50x20 matrix inline. This guarantees identical output across calls while keeping the source compact.

### Struct Design

Each domain defines its own struct types (`DemoVariant`, `DemoPeak`, `DemoSingleCell`, etc.) rather than using generic containers. This provides:

- Domain-specific field names matching standard bioinformatics conventions
- Type safety (e.g., `u64` for genomic positions, `f64` for p-values)
- Methods for format conversion (e.g., `to_vcf_line()`, `to_narrowpeak()`, `to_pdb()`)
- Analytical methods directly on the data (e.g., `shannon_diversity()`, `contact_map()`)

All structs derive `Debug` and `Clone`. Fields use `&'static str` for string data where possible to avoid allocations.

### Format Output Methods

Several structs include methods to produce standard bioinformatics format output:

- `DemoVariant::to_vcf_line()` produces tab-delimited VCF data lines
- `DemoPeak::to_narrowpeak()` produces BED6+4 narrowPeak lines
- `DemoPeak::to_bed()` produces BED6 lines
- `DemoStructure::to_pdb()` produces ATOM records with HEADER/END
- `demo_vcf_chr22()` produces a complete VCF file with header
- `demo_narrowpeak()` produces a complete narrowPeak file
- `demo_sdf_summary()` produces SDF-like records with `$$$$` delimiters
- `primate_nexus()` wraps a Newick tree in NEXUS format
- `cytochrome_b_fasta()` produces multi-sequence FASTA

These are intentionally simplified (e.g., the PDB output contains only Cα atoms) to serve as demo input for parsers in other cyanea crates.

### Analytical Methods

Some structs include basic analytical methods to make the datasets immediately useful without importing domain-specific crates:

- **`DemoSingleCell`**: `gene_expression()`, `total_counts_per_cell()`, `genes_per_cell()` for basic QC
- **`DemoOtuTable`**: `sample_totals()`, `relative_abundance()`, `shannon_diversity()` for basic ecological analysis
- **`DemoMolecule`**: `lipinski_compliant()`, `lipinski_violations()` for drug-likeness screening
- **`DemoStructure`**: `ca_distance()`, `contact_map()` for structural analysis

These methods are self-contained and do not depend on algorithms from other cyanea crates.

### Biological Realism

Datasets are designed to produce meaningful results when analyzed:

- **Single-cell**: T cell markers (CD3D, CD3E, IL7R) are enriched in T cells, B cell markers (CD19, MS4A1) in B cells, monocyte markers (CD14, LYZ) in monocytes. Clustering should recover the three groups.
- **Metagenomics**: IBD samples have lower Shannon diversity, depleted *Faecalibacterium* and *Ruminococcus*, and elevated *E. coli* and *C. difficile* — matching real IBD microbiome signatures.
- **Variants**: Mix of SNVs, a deletion (AGT->A), and an insertion (C->CTGA) with realistic allele frequencies from common 1000 Genomes variants.
- **Phylogenetics**: Distance matrix preserves the expected phylogenetic relationships (Human-Chimpanzee closest, Mouse Lemur most distant).
- **Structural**: Insulin chain B coordinates produce realistic Cα distances (~3.8 A between adjacent residues) and contact patterns.
- **Chemistry**: Drug library spans multiple therapeutic areas with a mix of Lipinski-compliant and non-compliant molecules.

### Dataset Sizes

All datasets are deliberately small to load instantly and display readably in notebooks:

| Dataset | Size | Use case |
|---------|------|----------|
| E. coli 16S | ~550 bp (truncated) | Sequence analysis demo |
| SARS-CoV-2 spike RBD | ~670 bp / ~222 aa | Alignment, variant analysis |
| chr22 variants | 10 records | VCF parsing, variant filtering |
| Gene annotations | 2 genes | Interval operations |
| Spike alignment seqs | 4 x 60 aa | MSA demo |
| Hemoglobin alpha | 5 x 141 aa | Cross-species alignment |
| ChIP-seq peaks | 10 peaks | Peak analysis, BED operations |
| CpG methylation | 12 sites | Methylation analysis |
| PBMC single-cell | 50 cells x 20 genes | Clustering, marker analysis |
| FDA drugs | 12 molecules | Cheminformatics screening |
| Primate tree | 12 species | Tree building, visualization |
| Distance matrix | 6 x 6 | NJ/UPGMA demo |
| Cytochrome b | 6 x 60 aa | Phylogenetic alignment |
| OTU table | 8 taxa x 6 samples | Diversity, differential abundance |
| Insulin chain B | 30 residues | Structure analysis, contact maps |
| Ramachandran angles | 10 points | Secondary structure assessment |

## Dependencies

```
cyanea-core (errors, traits)
```

No external dependencies beyond `cyanea-core`. All data is self-contained.

## Testing Strategy

- **Unit tests**: Inline `#[cfg(test)]` modules in each source file
- **Coverage**: Every public function and method is tested
- **Invariant checks**: Sequence validity (DNA bases only), matrix dimensions, symmetry of distance matrices, beta values in [0,1], summit within peak bounds, contact map symmetry
- **Biological assertions**: Marker enrichment in correct cell types, healthy > IBD diversity, Lipinski compliance counts, phylogenetic distance ordering
- **Format validation**: VCF header present, narrowPeak field count, FASTA record count, PDB ATOM line count
